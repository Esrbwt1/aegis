# Aegis Outreach Campaign Playbook v1

This document outlines the strategy and materials for the Phase 1b beachhead campaign, targeting the scientific and academic communities.

## 1. Target Audience Profiles

Our initial outreach will focus on individuals who are both computationally constrained and technologically curious.

*   **Profile A: The Graduate Student (Bioinformatics/Computational Physics):**
    *   **Pain Point:** Has complex simulations to run (e.g., protein folding, particle simulations) but must wait for days or weeks in a shared university HPC queue. Their research progress is directly tied to compute availability.
    *   **Motivation:** Highly motivated to find faster, cheaper, or more accessible ways to run their jobs to graduate sooner or publish more papers.

*   **Profile B: The Computer Science Student (Advanced/Hobbyist):**
    *   **Pain Point:** Interested in cutting-edge topics like decentralization, distributed systems, and Web3, but lacks a real-world project to apply their skills to.
    *   **Motivation:** Wants to contribute to a tangible, open-source project with a bold vision. Intrigued by the technical challenge.

*   **Profile C: The Independent Researcher / Small Lab PI:**
    *   **Pain Point:** Lacks the budget to access large-scale commercial cloud resources (like AWS/GCP). Their grant money is extremely limited.
    *   **Motivation:** Seeking radical cost-savings on essential computational tasks.

## 2. Core Message Template

This is a concise, adaptable message designed to capture attention and drive traffic to our GitHub repository.

**Subject/Title:** An open-source alternative to expensive cloud compute for scientific research?

**Body:**

> Hi [Name/Group],
>
> I'm part of an open-source project called Aegis, and we're building a decentralized network to provide low-cost access to computational power for scientific and academic use.
>
> We know that access to HPC resources can be a major bottleneck for research. Our goal is to create a global pool of shared compute so that no student or lab is held back by cost or queue times.
>
> We have just released our Alpha Protocol, a working prototype that can already execute tasks like Blender rendering and Python/Numpy data analysis across a mini-network. We are looking for feedback from the community that needs this most.
>
> Could you spare 5 minutes to check out our project on GitHub and see the vision? We have a one-click demonstration script that launches the entire network on a local machine.
>
> **GitHub Link:** https://github.com/Esrbwt1/aegis
>
> Any feedback, critique, or questions would be invaluable to us at this early stage.
>
> Thanks,
> Esrbwt1, on behalf of Project Aegis

## 3. Initial Communication Channels

We will start with channels where our target profiles congregate.

*   **Reddit:**
    *   r/HPC (High Performance Computing)
    *   r/bioinformatics
    *   r/PhysicsStudents
    *   r/compsci (in relevant threads about distributed systems)
    *   Post a modified version of the core message as a "Show and Tell" or discussion starter.

*   **University Forums / Discords:**
    *   Post in the general chat or project-sharing channels of the Unity University CS department and other relevant local universities.

*   **Direct Outreach (LinkedIn/Email):**
    *   Identify 5-10 local graduate students or professors from your university's computational science departments. Send them a personalized version of the core message.
